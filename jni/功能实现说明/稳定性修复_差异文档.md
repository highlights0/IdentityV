# 稳定性修复差异文档（旧行为 vs 新行为）

## 1. 差异总览
本差异文档记录 2026-03-25 的稳定性修复变更，重点针对“读取后卡死重启”风险。

## 2. 逐项差异

| 文件 | 位置 | 修改前 | 修改后 | 目的 |
|---|---|---|---|---|
| `src/Android_draw/draw_Gui.cpp` | 读取循环间隔 | `usleep(15000)` | `usleep(50000)`（`kReadLoopSleepUs`） | 降低驱动调用频率 |
| `src/Android_draw/draw_Gui.cpp` | 对象扫描上限 | 最多 2000 | 上限 600（`kMaxEntityScanCount`） | 降低单轮读取压力 |
| `src/Android_draw/draw_Gui.cpp` | 指针读取 | 多处直接 `getPtr64(...)` 链式嵌套 | `ReadPtr64Safe` 分步读取 + 逐层失败短路 | 防止野指针链式崩溃 |
| `src/Android_draw/draw_Gui.cpp` | 扫描页读取 | `vm_readv` 结果未检查 | `vm_readv` 失败则跳过当前页 | 防止脏数据参与偏移判断 |
| `src/Android_draw/draw_Gui.cpp` | 矩阵读取线程归属 | 渲染线程 `Draw_Main` 每帧直读驱动 | 读线程预取矩阵写入缓冲，渲染线程读缓存 | 减少每帧驱动访问 |
| `src/Android_draw/draw_Gui.cpp` | 对象坐标/标记读取 | 绘制路径里频繁读取 | 读线程预读缓存（`pos/flag_aa/is_ghost_obj`） | 降低渲染路径读压 |
| `src/Android_draw/draw_Gui.cpp` | 字符串写入 | 多处 `strcpy` | 改为 `snprintf` | 避免缓冲区越界风险 |
| `src/Android_draw/draw_Gui.cpp` | STB 实现宏 | 本文件也定义 `STB_IMAGE_IMPLEMENTATION` | 删除该定义，保留 `stb_image.cpp` 唯一实现 | 避免重复实现风险 |
| `src/Android_draw/kerneldriver-qxqd.hpp` | `getPtr64` | `driver->read` 失败也返回变量值 | `driver->read` 失败直接返回 `0` | 防止读取失败传播无效地址 |

## 3. 关键代码差异（摘录）

### 3.1 读取频率差异
- 旧：`usleep(15000);`
- 新：
```cpp
static constexpr long kReadLoopSleepUs = 50000;
...
usleep(kReadLoopSleepUs);
```

### 3.2 扫描上限差异
- 旧：
```cpp
if (count > 2000) count = 2000;
```
- 新：
```cpp
if (count > 2000) count = 2000;
if (count > kMaxEntityScanCount) count = kMaxEntityScanCount;
```

### 3.3 类名链读取差异
- 旧：单行多层嵌套解引用。
- 新：
```cpp
if (!ReadPtr64Safe(objPtr + Offsets::CLASSNAME_BASE, name_ptr_1) || name_ptr_1 == 0) continue;
if (!ReadPtr64Safe(name_ptr_1 + Offsets::CLASSNAME_STEP1, name_ptr_2) || name_ptr_2 == 0) continue;
if (!ReadPtr64Safe(name_ptr_2 + Offsets::CLASSNAME_STEP2, name_ptr_3) || name_ptr_3 == 0) continue;
if (!ReadPtr64Safe(name_ptr_3 + Offsets::CLASSNAME_STEP3, name_ptr_4) || name_ptr_4 == 0) continue;
if (!ReadPtr64Safe(name_ptr_4 + Offsets::CLASSNAME_STEP4, name_struct)) continue;
```

### 3.4 矩阵读取差异
- 旧：渲染线程中每帧通过 `getPtr64/vm_readv` 读取矩阵。
- 新：
```cpp
// read_thread
if (IsLikelyPtr(MatrixRealAddr) && vm_readv(MatrixRealAddr, buffer.matrix, sizeof(buffer.matrix))) {
    buffer.matrix_valid = true;
}

// Draw_Main
if (!buf.matrix_valid) return;
memcpy(matrix, buf.matrix, sizeof(matrix));
```

### 3.5 `getPtr64` 失败处理差异
- 旧：
```cpp
driver->read(addr, &var, 8);
return var & 0xffffffffffffff;
```
- 新：
```cpp
if (!driver->read(addr, &var, 8)) {
    return 0;
}
return var & 0xffffffffffffff;
```

## 4. 构建验证差异
- 旧：未做本次修复版本验证。
- 新：已执行 `ndk-build -C . NDK_OUT=./obj NDK_LIBS_OUT=./libs`，构建通过。

## 5. 结论
本次差异的核心是“减少高频无保护内核读 + 增加失败短路与缓存化”。
在你的实机反馈中，该版本已恢复正常可用，说明修复方向有效。
