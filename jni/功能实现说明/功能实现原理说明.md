# IdentityV 项目功能实现说明

## 1. 项目概览
该项目是 Android 原生层的叠加绘制程序，核心链路为：

驱动读内存 -> 特征扫描定位 -> 对象解析分类 -> 三缓冲同步 -> ImGui 叠加渲染

主入口在 `src/main.cpp`，启动后会守护化、创建 ANativeWindow 叠加层、初始化图形后端与触摸模块，并拉起数据读取线程。

## 2. 功能数据获取方式

### 2.1 目标进程定位
- 通过遍历 `/proc/*/cmdline` 匹配目标包名（`dwrg` / `com.netease.idv`）获取 PID。
- 对目标进程读取模块基址（`libclient.so` 或主模块 `.`）和大块 rw 内存段（BSS 区域候选）。

关键位置：
- `src/Android_draw/draw_Gui.cpp` 中 `get_name_pid1`、`get_module_base`、`get_module_bss`

### 2.2 内存读取通道
- 使用 `kerneldriver-qxqd.hpp` 中的 `c_driver` 封装。
- 通过 `/dev` 下字符设备节点 + `ioctl` 与内核驱动通信。
- 核心操作码：
  - `OP_READ_MEM = 0x801`
  - `OP_WRITE_MEM = 0x802`
  - `OP_MODULE_BASE = 0x803`

上层读取函数：
- `vm_readv`、`getDword`、`getFloat`、`getPtr64`

### 2.3 动态偏移定位（特征扫描）
读取线程会先在候选内存段扫描特征，避免写死地址：

- 矩阵特征：`0x656A624F72655028`，并校验 `+304 == 257`，得到 `MatrixOffset`
- 数组特征：值 `16384`，并校验相邻 `float == 1.0`、`dword == 257`，得到 `ArrayaddrOffset`

说明：偏移常量在 `src/Android_draw/Offsets.h` 中维护，便于版本更新。

### 2.4 对象数据解析
定位对象数组后，按元素读取对象指针并解析：

1. `objPtr = *(StartPtr + i*8)`
2. 坐标组件指针：`component_ptr = *(objPtr + 0x28)`
3. 类名链：`obj + 0xF8 -> 0x0 -> 0x8 -> 0x20 -> 0x20 -> name_struct`
4. 名字长度与字符串地址：`name_struct + 0x10`、`name_struct + 0x8`

根据类名关键词分类：
- 监管者：`boss`
- 求生者：`player`
- 场景（密码机）：`scene_sender + low`
- 地窖：`prop_76`
- 道具：`prop`

角色显示名由 `src/Android_draw/Name.h` 的映射表转换（`getplayer/getboss`）。

### 2.5 并发与缓冲
使用三缓冲 + 原子索引避免读写冲突：
- `g_buffer[3]`
- `g_read_index`
- `g_write_index`

读取线程约每 `15ms` 刷新一次数据，渲染线程只读当前 `g_read_index` 指向的缓冲。

## 3. 功能实现原理

### 3.1 叠加层创建与渲染循环
- 通过 `ANativeWindowCreator` 使用 `SurfaceComposerClient` 创建系统叠加窗口。
- 图形后端默认 OpenGL ES 3（也预留 Vulkan 实现）。
- 每帧流程：
  1. `drawBegin()`（屏幕方向与窗口状态更新）
  2. `graphics->NewFrame()`
  3. `Layout_tick_UI()`（绘制主逻辑 + 控件）
  4. `graphics->EndFrame()`

### 3.2 世界坐标转屏幕坐标（W2S）
`Draw_Main` 每帧读取相机矩阵并做投影：

- 深度：`cam_z = M3*x + M7*z + M11*y + M15`
- 屏幕 X：`r_x = px + (...) / cam_z * px`
- 屏幕 Y：`r_y = py - (...) / cam_z * py`

基于投影结果绘制：
- 人物方框
- 名字与距离
- 可选射线
- 调试类名与坐标文本

### 3.3 自身锁定策略
不是固定读取“本地玩家指针”，而是屏幕中心候选法：
- 在阵营对象中筛选深度范围（`cam_z`）
- 读取标志位做有效性过滤
- 选取 `|r_x - px|` 最小且小于阈值的目标作为“自身”

作用：在部分视角变化场景下提升稳定性。

### 3.4 过滤与有效性判定
`ShouldSkipEntity` 负责过滤无效对象：
- 高度阈值过滤（如 `pos.Z <= -300`）
- 类名关键词过滤（特效、阴影、宠物、诱饵等）
- 通过校验值判断“幽灵/异常对象”
  - `obj + 0x70` 与 `VALID_ACTOR_VALUE`
  - `obj + 0x1A0` 与 `VALID_ACTOR_FLOAT`

### 3.5 小地图功能
- 贴图来源：`/sdcard/maps/{index}.png`
- 先按地图配置（`Maps[]`）把世界坐标归一化，再映射到小地图像素坐标
- 分层绘制：密码机、地窖、监管者、求生者、自身朝向箭头

### 3.6 输入与 UI 控制
- 触摸由 `TouchHelperA` 读取 `/dev/input/event*`，并映射到 ImGui 鼠标事件。
- 音量键监听线程：
  - 音量减：隐藏窗口
  - 音量加：显示窗口

UI 中可切换：
- 射线、名字/距离、密码机/地窖、道具、幽灵过滤、调试模式、小地图参数等。
